## 二、发现的问题及修复

### 2.1 严重问题（已修复）

#### 问题 1: 排序逻辑错误
**文件**: `TSSportVC.m` (第136-138行)
**问题描述**: 排序比较器在相等情况下返回错误的结果
**影响**: 可能导致运动数据排序不稳定
**修复方案**:
```objc
// 修复前
return obj2.summary.startTime > obj1.summary.startTime ? NSOrderedDescending : NSOrderedAscending;

// 修复后
if (obj2.summary.startTime > obj1.summary.startTime) {
    return NSOrderedAscending;
} else if (obj2.summary.startTime < obj1.summary.startTime) {
    return NSOrderedDescending;
}
return NSOrderedSame;
```

#### 问题 2: 数组越界风险
**文件**: `TSSportVC.m` (第168、187行)
**问题描述**: 访问数组元素前未检查索引有效性
**影响**: 可能导致应用崩溃
**修复方案**: 在访问数组前添加边界检查
```objc
if (indexPath.row < _sportList.count) {
    TSSportModel *sport = _sportList[indexPath.row];
    // ...
}
```

#### 问题 3: 配速显示条件不完整
**文件**: `TSSportDetailVC.m` (第189行), `TSSportCell.m` (第170行)
**问题描述**: 只检查 avgPace > 0，未检查 distance > 0
**影响**: 无距离的运动（如瑜伽）可能显示错误的配速
**修复方案**:
```objc
// 修复前
if (summary.avgPace > 0)

// 修复后
if (summary.avgPace > 0 && summary.distance > 0)
```

#### 问题 4: 心率图表除零风险
**文件**: `TSHeartRateChartView.m` (第93、158行)
**问题描述**: 当所有心率值为0或相等时可能导致除零错误
**影响**: 可能导致应用崩溃或显示异常
**修复方案**:
- 添加心率值有效性检查（hrValue > 0）
- 为全0数据设置默认范围（60-100 bpm）
- 跳过无效的心率值

#### 问题 5: 折线图单点数据除零
**文件**: `TSHeartRateChartView.m` (第147行)
**问题描述**: 当只有一个数据点时，spacing 计算会除零
**影响**: 可能导致应用崩溃
**修复方案**: 为单点数据添加特殊处理逻辑

#### 问题 6: 数据采样丢失最后一个点
**文件**: `TSHeartRateChartView.m` (第223-240行)
**问题描述**: 采样算法可能丢失最后一个数据点
**影响**: 图表显示不完整
**修复方案**:
```objc
// 确保包含第一个和最后一个数据点
[sampledData addObject:data[0]];
// 采样中间点...
[sampledData addObject:data[data.count - 1]];
```

#### 问题 7: 心率颜色计算缺少边界检查
**文件**: `TSHeartRateChartView.m` (第242-258行)
**问题描述**: 当 maxHrValue 为 0 时，所有颜色判断失败
**影响**: 图表颜色显示不正确
**修复方案**:
- 添加 maxHrValue > 0 的检查
- 提供基于通用心率区间的备用颜色方案

---

### 2.2 次要问题（已修复）

#### 问题 8: 魔法数字过多
**文件**: `TSSportDetailVC.m` (第85行)
**问题描述**: 心率卡片高度计算使用硬编码数字
**影响**: 代码可维护性差
**当前状态**: 已添加注释说明各部分高度，建议后续重构为常量
